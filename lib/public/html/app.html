Instructions:

Welcome to Railsonify, an experiment in the sonification of rail route and timetable data.

Note to the user: This application has been tested using Firefox & Chrome on MacOS and Ubuntu. However, audio issues such as popping and glitching do still occur, mainly due to a slowing/blocking caused by the inefficiency of browsers on some systems. Apologies for this.

How to use:

This application is semi-interactive, meaning you have control over the monitored stations. To do this, enter a station code (a three character format string) or a location (any town/city will work) to set a montitored station. The number of stations you may monitor at any one time is technically limited only by your browser and the API pipeline, so multiple station entries will begin a simultaneous monitoring of multiple stations.

Sounds are generated in two different formats/modes: Discrete and Continuous (for further information on the differences between these methods, please see the relevant sections in the accompanying report).
